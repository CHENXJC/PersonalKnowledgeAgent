# Demo Data Guide

PersonalKnowledgeAgent is a personal knowledge base project. Public versions
must never include the real private knowledge base.

## Approved Demo Content

Use notes that are fictional, generic, or manually sanitized. Safe examples:

- A fictional AI project summary.
- A generic university marketing note.
- A portfolio positioning note without real application records.
- A small business automation idea with no customer data.
- A public-style market observation with no financial account information.
- A long-term goal note with no private location, identity, or contact details.

## Disallowed Content

Do not include:

- Real private notes copied from the user knowledge base.
- Real assignment files or private university records.
- Real job applications, resumes, contracts, or contact information.
- Real customer, student, employer, or client data.
- Secrets, API keys, tokens, passwords, recovery codes, or credentials.
- Passport, visa, bank, tax, medical, or identity documents.
- Raw logs or generated outputs that were not manually reviewed.

## Demo Knowledge Base

The safe demo folder is:

```text
demo_knowledge_base/
```

It is separate from the real local knowledge base and can be used for future
screenshots or a public demo mode.

## Sanitization Checklist

- Replace real names with role labels such as `Demo Student` or `Demo Project`.
- Remove emails, phone numbers, addresses, IDs, and account references.
- Remove private file paths.
- Remove dates that identify private events unless they are generic.
- Keep notes short and clearly marked as demo content.
